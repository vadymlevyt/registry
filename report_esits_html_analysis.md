## 2. Виявлені особливості ЄСІТС HTML формату

### Формат 1: ЄСІТС legacy (6 з 7 файлів)

Це **Word "save as HTML"-style** з кастомними CSS класами що генеруються сервером ДСА (Державна судова адміністрація).

**Кодування:** `<meta http-equiv="Content-Type" content="text/html; charset=Windows-1251">`

**Структура:**

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" ...>
<html><head>
  <META NAME="FILENAME" CONTENT="...">
  <META NAME="CAUSENUM" CONTENT="...">
  ... 20-100+ META тегів з даними справи ...
  <title>...</title>
  <meta http-equiv="Content-Type" content="text/html; charset=Windows-1251">
  <style type="text/css"><!--
    body, table, span.rvts0 { font-family: 'Times New Roman'... }
    span.rvts1 { font-weight: bold; color: #0000ff }
    ...
    .rvps1 { text-align: center }
    .rvps7 { text-align: justify; text-indent: 36px }  ← основний абзац!
    --></style>
</head>
<body>
  <p class=rvps3><img width=54 height=73 alt="" src="data:image/png;base64,..."></p>
  <p class=rvps4><span class=rvts15>У Х В А Л А</span></p>
  <p class=rvps7><span class=rvts13>...основний текст справи...</span></p>
  <p class=rvps2><span class=rvts15>Суддя:</span> ... <span class=rvts15>І.П. Прізвище</span></p>
</body></html>
```

**CSS класи — два префікси:**

- `rvts*` — **text styles** (run/span styles): bold, italic, color, font-size, background. Завжди застосовуються через `<span class=rvtsN>`.
- `rvps*` — **paragraph styles**: text-align, margin-left, text-indent. Через `<p class=rvpsN>` або `<div class=rvpsN>`.

**Стандартні rvps* для адвокатських документів:**

| Клас | text-align | Інше | Призначення |
|------|------------|------|-------------|
| (default `p`) | left | indent 0 | Звичайний абзац |
| `rvps1` | center | — | Заголовки («У Х В А Л А», секції) |
| `rvps2` | right | — | Номер справи, дата+місто (шапка справа) |
| `rvps3` | center | margin-left:48px | Герб або вкладений заголовок |
| `rvps4` | left | margin-left:48px | Вкладений вступ |
| `rvps5` | center | text-indent:36px | Центрований з відступом |
| `rvps6` | left | text-indent:36px | Звичайний з абзацним відступом |
| `rvps7` | **justify** | text-indent:36px | **Основний текст** документа |

⚠ **Класи rvtsN/rvpsN НЕ нормалізовані — нумерація залежить від файлу.** У одній ухвалі `span.rvts15 { font-weight: bold }`, а в іншій `span.rvts15 { display: none }`. Тому парсер ЗОБОВ'ЯЗАНИЙ читати `<style>` блок кожного конкретного файлу — не може хардкодити.

### Формат 2: ВКЗ2 modern HTML5 (1 з 7 файлів)

Підсистема Відеоконференцзв'язку — окремий генератор. UTF-8, Bootstrap, modern HTML. Без специфічних ЄСІТС класів. Рендериться відносно нормально без додаткових фіксів.

## 3. Спільне і відмінне у різних типах документів

### Спільне (всі 6 ЄСІТС legacy):

1. Charset Windows-1251 у meta http-equiv
2. CSS у `<style>` блоці з `rvts*` (span) і `rvps*` (paragraph)
3. Селектори без тегу: `.rvps1 { ... }`
4. Кома-розділені селектори: `body, table, span.rvts0 { ... }`
5. Атрибути без лапок: `class=rvts11`, `width=54`, `height=73`
6. Текст у `<span class=rvtsN>` всередині `<p class=rvpsN>`
7. Базовий шрифт Times New Roman 12pt через `body, table, span.rvts0`

### Відмінне:

| Тип | Особливість |
|-----|-------------|
| **Ухвала** | Герб у `<p class=rvps3>`, заголовок «У Х В А Л А» центрований жирним, основний текст `rvps7` justify, підпис справа |
| **Ухвала апеляції** | Герб у `<p class=rvps2>` (правий край), додатково META `PREVCOURTNAME`, `PREVCAUSECOURT` |
| **Ухвала 2-о (без розгляду)** | META має `REREGCAUSENUM1`, `REREGCAUSEDATE1` (перереєстрація). Спеціальний індекс «2-о» |
| **Повістка** | 2-колонкова `<table border=1>` з суд/особа. META `DOCTYPE=7`. Складні `<span>` chain для адрес |
| **Фіксація розподілу** | Багато `rvps3`, `rvps4`, `rvps5`, ... з різними margin. Спеціальні маркери `<!--BEGIN_CAUSE_MEMBERS-->` |
| **dnzs (наказ)** | Спрощений: тільки 18pt заголовок, 14pt subheaders, без герба. ~20 META |
| **Протокол ВКЗ2** | UTF-8, Bootstrap, video tags. Окрема система |

### Особливі елементи:

- **Герб** — однаковий PNG 54×73px у всіх ухвалах (data: URI з тим самим base64 ~1.2 КБ). Це константа сервера ДСА — стилізоване зображення тризубу.
- **Таблиці** — у повістці (2 колонки 338px кожна, border=1, valign=top без лапок).
- **`<!--BEGIN_CAUSE_MEMBERS-->`** — маркер у фіксації розподілу. Між ним і `<!--END_CAUSE_MEMBERS-->` — список учасників. **Потенційно корисно** для майбутнього модуля автоімпорту учасників.
- **`&nbsp;` в shaping** — багато non-breaking spaces для вирівнювання. Парсер нормалізує у звичайний пробіл (`\s+` regex матчить U+00A0).

## 4. Причина чому герб не рендерився

**Корінь:** `<img>` всередині `<p class=rvps3>` додавався мoїм `collectInlineRuns` як inline run типу `'image'`. Далі `layoutLines` ставив image у окремий рядок як segment. Але **`drawLine` обробляв ТІЛЬКИ `seg.type === 'word'` і `seg.type === 'space'`** — image-сегменти ігнорувались.

```js
// БУЛО: drawLine
for (const seg of segs) {
  if (seg.type === 'space') { x += seg.width; continue; }
  if (seg.type === 'word') { ... drawText ... }
  // seg.type === 'image' — НЕ ОБРОБЛЯЛОСЬ → герб ніколи не рендерився
}
```

## 5. Які виправлення зроблено у парсері

### Фікс А: `splitParagraphByImages`

Замість того щоб збирати img як inline run всередині параграфа, walkDom тепер викликає `splitParagraphByImages` для `<p>` і `<center>`. Функція ітерує дітей параграфа і нарізає його на послідовність блоків:

```
[paragraph до img] [image block] [paragraph після img] [image block] ...
```

Якщо параграф містить **тільки** img + порожні `<span>` (як у ЄСІТС шапці) — створюється **тільки** image block, без порожнього paragraph.

### Фікс Б: `parseImgLength` для атрибутів width/height

HTML4 specifies: `<img width=54 height=73>` без юніту = пікселі. Старий `parseLength` трактував `''` як pt (CSS-style) — давав 54pt замість 54px (~ 72pt). Новий хелпер `parseImgLength` конвертує: integer без юніту → px → pt (×0.75).

### Фікс В: дедуплікація герба

Set `seenImageSrcs` тепер передається через walkDom + splitParagraphByImages. Один герб не рендериться двічі (Word conditional comments + VML + img fallback теоретично можуть давати дублікати — раніше були проблемою тільки для іншого формату Word).

### Підтверджено що працює:

- **Селектори без тегу** (`.rvps7 { text-align: justify }`) — мій `parseStyleBlock` обробляє `tag=null, cls='rvps7'`.
- **Кома-розділені селектори** (`body, table, span.rvts0`) — розбиваються на 3 окремі rules.
- **Атрибути без лапок** (`class=rvts11`) — DOMParser браузера і jsdom обробляють нативно.
- **`<style>` блок у `<head>`** — `collectStyleSheet(doc)` читає всі `<style>` через `doc.getElementsByTagName('style')`.
- **META теги** — пропускаються walkDom через SKIP_TAGS, але залишаються в DOM. Доступні через `doc.querySelectorAll('meta[name]')` для майбутнього модуля ЄСІТС.

## 6. Зведена таблиця META даних для майбутнього модуля ЄСІТС

Майбутній модуль автоматичного імпорту справ з ЄСІТС зможе зчитувати ці поля з кожного HTML файлу. Готовий API.

### Загальні поля справи (у всіх ухвалах і повістках):

| Поле | Опис | Приклад |
|------|------|---------|
| `FILENAME` | Технічна назва файлу на сервері | `1005_22502358.html` |
| `DOCID` | Унікальний ID документа в ЄСІТС | `22502358` |
| `DOCORGID` | ID організації-генератора | `1036` |
| `COURTNAME` | Повна назва суду | `Бориспільський міськрайонний суд...` |
| `CAUSECOURT` | Код суду в реєстрі | `1005` |
| `CAUSECOURT2013` | Старий код суду до 2013 | `359` |
| `CAUSENUM` | **Номер справи** | `359/12899/25` |
| `CAUSEDATE` | Дата надходження справи | `30.10.2025` |
| `PROCNUM` | **Номер провадження** | `1-кп/359/901/2025` |
| `DOCDATE` | Дата документа | `30.10.2025` |
| `DOCTYPE` | Тип документа (number) | `4` (ухвала), `7` (повістка) |
| `CAUSETYPE` | Тип справи | `2` (кримінальна), `4` (цивільна) |
| `CAUSECATEGORY` | Категорія за класифікатором | `411010714` |
| `CAUSELOCATION` | Локація: `P` (parent), `A` (appellate) | `P` |

### Дані судді (1+):

| Поле | Опис |
|------|------|
| `JUDGEPOSITION{N}` | Порядковий номер судді (1, 2, 3...) |
| `JUDGENAME{N}` | ПІБ судді |
| `JUDGECODE{N}` | Унікальний код судді |

### Дані учасників (1..N):

| Поле | Опис | Приклад |
|------|------|---------|
| `MEMBPOSITION{N}` | Порядковий номер | `1` |
| `MEMBTYPE{N}` | Тип: P (позивач), V (відповідач) тощо | `V` |
| `MEMBPOZYVACHTYPE{N}` | Підтип позивача (L=юр, F=фіз...) | `L` |
| `MEMBVIDPOVIDACHTYPE{N}` | Підтип відповідача | `G` |
| `MEMBOSOBATYPE{N}` | Тип особи: F (фіз), U (юр) | `F` |
| `MEMBNAME{N}` | **ПІБ повне** | `Ткаченко Сергій Васильович` |
| `MEMBSHORT{N}` | Скорочений підпис | `Відповідач 1` |
| `MEMBOKPO{N}` | **РНОКПП фіз / ЄДРПОУ юр** | `2958638797` / `04527520` |
| `BIRTDATE{N}` | Дата народження (для фіз. осіб) | `13.02.1963` |
| `MEMBPOSTADDRESS{N}` | Поштова адреса | `вул. Лісовицька, ...` |
| `MEMPHONE{N}` | Телефон | `380633131564` |
| `MEMDOCTYPE{N}` | Тип документа особи | `Паспорт громадянина України` |
| `CHASTDERGVL{N}` | Доля держави (так/ні) | `FALSE` |

### Спеціальні поля:

- **Ухвала апеляції:** додатково `PREVCOURTNAME`, `PREVCAUSECOURT`, `PREVCAUSECOURT2013`, `PREVCAUSECATEGORY` — попередня інстанція.
- **Ухвала 2-о:** додатково `REREGCAUSENUM{N}`, `REREGCAUSEDATE{N}` — повторна реєстрація.
- **Повістка з кількома адресами:** `REREGCAUSENUM1..N` — список перереєстрацій.

### Технічні поля (для аудиту/підпису):

- `APPVER` — версія генератора (наприклад `3.32.4.501`)
- `CRYPTPROVIDER` — крипто-провайдер ЕЦП (`DS13`)
- `PRP_*` — номер електронного підпису
