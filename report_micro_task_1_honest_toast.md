# Мікро-TASK 1 — Чесний тост після "Розпізнати зараз"

**Дата:** 2026-05-09
**Статус:** Виконано

## Що зроблено

Тост після натискання "Розпізнати зараз" у Viewer тепер чесно відображає реальний результат запису кеша на Drive, а не лише факт успішного виклику OCR. Раніше адвокат бачив зелений тост `"Текст оновлено"` навіть коли файл фактично не з'являвся в `02_ОБРОБЛЕНІ` (через `skipCache:true`, який блокує `writeCache`, або потенційні помилки upload). Тепер `extractText` повертає поле `cacheWritten: boolean`, яке відображає реальний результат `writeCache`. У `onReprocess` тост розгалужується: `success` коли кеш фактично записаний, `warning` коли OCR спрацював, але кеш не зберігся.

## Що змінилось у файлах

### `src/services/ocrService.js`

- **`writeCache`** (рядки 93-110) — тепер повертає `boolean`:
  - `true` — `uploadTextFile` завершився без помилок
  - `false` — `subFolderId` відсутній, або `try/catch` зловив помилку (зокрема upload failures, які раніше мовчали)
- **`extractText`** (рядки 167-240):
  - Cache-hit branch (рядки 175-184) → додано `cacheWritten: false` для консистентності shape
  - Provider success branch (рядки 211-225) → локальна змінна `let cacheWritten = false`, потім `cacheWritten = await writeCache(...)` коли `!options.skipCache`, і `cacheWritten` пробрасується в return
  - Коли `options.skipCache === true` (як у reprocess) → `writeCache` не викликається, `cacheWritten` залишається `false` — це коректно відображає реальність

### `src/components/CaseDossier/index.jsx`

- **`onReprocess`** (рядки 2448-2462):
  - `await ocrService.extractText(...)` тепер захоплюється у локальну `ocrResult`
  - Розгалуження за `ocrResult?.cacheWritten`:
    - `true` → `toast.success('Текст розпізнано і збережено')`
    - `false` → `toast.warning('Текст розпізнано, але не вдалось зберегти кеш на Drive', { description: 'При повторному відкритті може знадобитись повторне розпізнавання' })`

## Як перевірити

**Що очікувати при поточному стані коду:** натиск на "Розпізнати зараз" завжди виводитиме **жовтий warning-тост** `"Текст розпізнано, але не вдалось зберегти кеш на Drive"`. Це **очікувана поведінка цього мікро-TASK** — він робить тост чесним, але не виправляє причини того, що кеш не записується.

**Сценарій тестування:**

1. Відкрити будь-який документ адвоката (PDF) у CaseDossier через DocumentViewer.
2. Перемкнути на режим "Текст" — побачите empty state з кнопкою "Розпізнати зараз".
3. Натиснути "Розпізнати зараз".
4. Чекати ~5-15 сек поки відпрацює OCR.
5. **Очікуваний результат:** жовтий warning-тост `"Текст розпізнано, але не вдалось зберегти кеш на Drive"` з підказкою про повторне розпізнавання.
6. **НЕ очікується:** зелений success-тост (бо кеш фактично не пишеться, і це чесно показано).

Коли в наступному мікро-TASK буде виправлено `skipCache:true` (щоб writeCache писав і при reprocess) і кириличну `q=` filter — тост стане **зеленим** автоматично. Тобто адвокат зможе візуально верифікувати фікс просто за кольором тоста.

## Що НЕ зроблено і чому

Цей мікро-TASK свідомо обмежений лише чесним тостом. **Не виправлені** і чекають окремих мікро-TASK-ів:

- **Мікро-TASK 2 — кирилиця в `q=` (правило #8) у `ocrService.js`:**
  - `listFolderFilesByName` (рядок 45-49) використовує кириличну назву файла в `q=` параметрі Drive API.
  - `cacheFileName` будує ім'я з `originalName`/`name` документа, які є кириличні.
  - `writeCache` і `checkCache` через це працюють ненадійно, навіть коли потрапляють у виклик.
  - Окремо також: `skipCache:true` у `onReprocess` блокує сам виклик `writeCache`. Зараз reprocess нічого не пише — варто перепроектувати семантику `skipCache` або додати окремий `forceRefresh` прапор.

- **Мікро-TASK 3 — pipeline створення копії в `02_ОБРОБЛЕНІ` після завантаження через `AddDocumentModal`:**
  - `onSubmit` модалки (CaseDossier:2804-2849) завантажує файл лише в `01_ОРИГІНАЛИ` через `uploadFileLocal`, далі робить `createDocument` і `add_document`.
  - Жодного виклику `ocrService.extractText` чи копіювання в `02_ОБРОБЛЕНІ` немає.
  - Pipeline (текстовий PDF: одразу OCR → запис тексту в `02_ОБРОБЛЕНІ`; сканований: OCR через documentAi → запис) треба додати окремо.
