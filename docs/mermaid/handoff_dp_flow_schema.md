## 2. ДЕ ЗАРАЗ DP (станом на 2026-05-29, після TASK 1)

**TASK 1 (image_merge_unify) — ЗАВЕРШЕНО** (1A + 1C + 1B + crop-фікс):
- **1A** — reusable винесено у `components/ImageEditor/` + `services/imageDocument/`.
- **1C** — детермінований вибір маршруту + тумблер «Просто додати файли» (skipPdfSlicing)
  + warning-fix. `.txt` для text-layer PDF свідомо НЕ пишемо (самодостатній шар).
- **1B** — N-документна склейка фото в DP: вибір сценарію НА ВХОДІ (фото обходять
  PDF-OCR), `imageDocumentGrouper` (Haiku) розкладає фото по документах, редактор у
  Зоні 3 (drag між групами, crop/rotate), «Виконати» → N окремих PDF.
- **crop-фікс** — сітка показує обрізані фото так само як попап.

**Що працює реально:**
- Фото → N документів з правкою плану (Ітерація 1 для image-merge ПОВНА).
- Нарізка PDF (для 1-3 файлів, ≤70 стор. добре).
- Тумблер «Просто додати файли».

**Що ще НЕ працює (візія, не реальність):**
- Правка плану для **нарізки PDF** — досі autoConfirm (Фаза 5, найбільша прогалина).
- Більшість modifier'ів Зони 2: organizeByProceedings, integrityCheck, generateSummary,
  suggestDeadlines, updateCaseContext, fillCaseCard — у UI є, логіки нема.
- ZIP unpack (код є, не підключено), RTF/ODT.

## 3. ВІДКРИТІ ПИТАННЯ (з цієї сесії — у consultation_dp_flow_observations.md)

**#1 — Тумблер «Просто додати файли» оминає правку плану.**
Адвокат помітив на схемі: тумблер вів через крок правки плану — це суперечить його
сенсу. Виправлено в схемі (тумблер → OCR → підсумок → у справу, без плану).
⚠️ Врахувати на **Фазі 5**: коли зʼявиться реальний гейт правки плану нарізки —
тумблер ON має його **оминати**.

**#2 — Стискання, можливо, ПЕРЕД OCR, а не в кінці.**
Скани надлишкової якості (600 dpi); стискання на вході полегшило б OCR (ліміт ~40 МБ,
швидкість, RAM iPad). Золота середина ~200-300 dpi. Потрібен **замір** на реальних
сканах перед рішенням (стискати робочу копію, оригінал зберігати повним). Тригер —
TASK стискання / якості великих томів.

## 4. ТЕМИ ДЛЯ ПОДАЛЬШОГО ПОГЛИБЛЕННЯ (адвокат сказав «можливо ще будуть питання»)

Ймовірні напрямки які варто простежити по схемі:
- **Мікс фото + PDF** — зараз свідома межа (toast + борг #27). Як має виглядати
  правильний мікс-флоу у візії?
- **Гейт правки плану для нарізки** (Фаза 5) — як саме адвокат правитиме дерево
  документів великого тому (drag-merge/split сторінок)?
- **Modifier'и** — порядок застосування, які до основної дії, які після (стискання — кейс #2).
- **Ітерація 3** (AI-оркестратор над сценаріями) — коли і як AI пропонуватиме
  комплексний план для міксу.
- **case_context / картка справи** — як DP живить досьє (перетин з TASK 2).
