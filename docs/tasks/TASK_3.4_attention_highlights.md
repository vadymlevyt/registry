## МЕТА ФАЗИ

AI при очистці **підсвічує в самому тексті** підозрілі фрази (можлива розбіжність / дивне
формулювання / схоже на OCR-помилку), **не змінюючи зміст**. Адвокат бачить жовте прямо в
документі, має список цих місць із навігацією, може сховати підсвітки або зняти назавжди.

**Видимий результат:** почищений `.md` має жовті підсвітки; у в'ювері (режим Текст) — чип
«⚠ N поміток» → панель зі списком, навігацією (клік → скрол до місця), перемикачем показу і
«зняти всі назавжди».

## PHILOSOPHY CHECK
- **Прив'язка до тексту, не до крихкого номера** — мітка стоїть на словах (AI сам їх обгортає
  при генерації) → немає проблеми зсуву пачок / номерів сторінок.
- **Зміст недоторканий (#6 design)** — мітка `==…==` форматна; слова всередині дослівні.
- **Один вхід (#11 + мердж UI)** — чип → панель з усім; не плодимо кнопки.
- **Розширюваність** — дизайн навмисно готовий до per-mark редагування (борг #47), без рефактору.

## СКЛАДОВІ

### 3.4.1 — Інлайн-мітки в ядрі (промпт + форма нотатки)
- `buildCleanPrompt`: додати правило — підозрілу фразу **обгорнути** `==фраза==`, **дослівно**
  (форматна позначка, не зміна змісту); і **продублювати** її в `attentionNotes` (короткий
  фрагмент + причина). Делімітер `==` (або колізійно-безпечний `‹‹…››` — обрати, звірити що не
  ламає GFM-таблиці/код).
- `attentionNotes`: **прибрати `page`** → форма `{ note }` (фрагмент+причина) у тому ж порядку,
  що мітки в тексті. Оновити parse в `polishToMarkdown`, схему extended, тести 3.1.
- Скоуп-гард scanned лишається (мітки лише на сканованих, бо очистка лише там).

### 3.4.2 — Рендер міток (MarkdownRenderer)
- `inline()`: `==x==` → `<mark class="attention" data-mark="N">x</mark>`, де `N` — **порядковий**
  індекс мітки в документі (1,2,3…). Робити на ВЖЕ екранованому тексті (XSS-safe, як `**`/`*`).
- Колір — **новий design-токен** `--attention-bg` (жовтий/янтарний; parent §СПІЛЬНІСТЬ ДИЗАЙНУ),
  НЕ hex по місцю.

### 3.4.3 — Чип + панель у в'ювері (лише режим Текст)
Дизайн (узгоджено):
```
┌─ в'ювер, режим ТЕКСТ (хедер/футер, біля «Очистити текст») ─┐
│  [⚠ 5 поміток]   ← ЄДИНИЙ елемент; у режимі Скан відсутній  │
└────────────────────────────────────────────────────────────┘
        │ тап
        ▼
┌─ панель поміток ───────────────────────────────────────────┐
│  [✓] Підсвічувати в тексті        ← вкл/викл (CSS, миттєво)  │
│  ───────────────────────────────────────────────────────── │
│  1. «сума в розписці…»   — звір з оригіналом            →   │  ← клік = скрол+пульс
│  2. «дата 12.03.2024…»   — можлива OCR-помилка          →   │
│  ───────────────────────────────────────────────────────── │
│                 [ Зняти всі назавжди ]                       │
└──────────────────────────────────────────────────────────────┘
```
- **Прив'язка:** чип рендериться лише `viewMode === 'text' && markCount > 0`. Режим Скан — чипа
  немає (мітки — властивість текстового рендеру).
- **Мердж кнопок:** один вхід-чип; усередині — перемикач показу + список + «зняти всі». На тулбарі
  рівно одна штука.
- **Список:** пункт ↔ `data-mark` за **порядком** (1↔1). Джерело: мітки з `.md` + причини з
  `extended.attentionNotes` (обидва persisted у 3.1) → працює і «пізніше».
- **Клік-навігація:** `element.scrollIntoView({behavior:'smooth', block:'center'})` + тимчасовий
  клас «пульс». Ми володіємо рендером (`.md` ≠ Mozilla-PDF) — тривіально.
- **Перемикач показу:** CSS-клас на контейнері ховає/показує `<mark>` (миттєво, нічого не зберігаємо).
- **«Зняти всі назавжди»:** стрипнути всі `==` з `.md` (зберегти текст), re-save через
  `cleanTextDriveAdapter.saveMarkdown`, очистити `attentionNotes` в extended → чип зникає.
- **Колір:** жовтий/янтарний = «увага» (НЕ зелений).
- **Спільні хелпери** (parent §СПІЛЬНІСТЬ): парсинг міток, стрип `==`, scroll-to-mark — переюзні
  (кнопки, майбутній агент-менеджер #47, редактор) → не локальні копії.
