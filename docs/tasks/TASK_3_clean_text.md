## МЕТА (загальна)

Перетворити сирий OCR-текст сканованого документа (`.txt`) на **гарний читабельний
Markdown** (`.md`) — абзаци, заголовки, жирний шрифт, таблиці, близько до оригіналу —
**не змінюючи юридичний зміст**. Зробити це **спільним сервісом з народження**
(Rule of Three / #11), що його кличуть кілька точок. Паралельно винести механіку
вибору файлів з Архіву у спільний компонент для пакетних операцій у реєстрі.

## СКОУП (наскрізний, перевірено по коду — критично для ВСІХ фаз)

Очистка працює **ВИКЛЮЧНО** для `documentNature==='scanned'` (скани/фото через Document AI —
мають `.layout.json`/`mergeLayout`). **`searchable` (DOCX/HTML/текстовий PDF) — ПОВНІСТЮ
ПОЗА ФУНКЦІЄЮ** (кнопка неактивна / цикл пропускає).

Причина (по коду): `.layout.json` створює ТІЛЬКИ Document AI; `pdfjsLocal` за дизайном
не дає `pageStructure` (`ocr/pdfjsLocal.js:8`). У searchable вже чистий цифровий текст —
чистити нема чого.

| Формат | documentNature | `.layout.json` | Очистка |
|--------|---------------|----------------|---------|
| Скан PDF/фото (Document AI) | `scanned` | ✅ | ✅ |
| Фото склеєні→PDF (DP) | `scanned` | ✅ | ✅ |
| DOCX / HTML / текстовий PDF | `searchable` | ❌ | ❌ ігнор |

## СПІЛЬНІ АРХІТЕКТУРНІ РІШЕННЯ (наскрізні)

### Гібрид 3 кроки (Вісь A, реалізується у 3.1, переюз у 3.2)
1. **Конденсатор** (детермінований, 0 токенів AI): layout → компактна Markdown-чернетка.
   Читає **ПОСТОРІНКОВО через `page._text`** (offset'и в глобальний `.txt` ненадійні на
   сканах >25 стор. — `documentAi.js:428-435` перебазовує лише `pageNumber`). Структуру
   виводить з геометрії `boundingPoly`+розмір. Дзеркало `pageMarkers.js:80-128`.
2. **AI-поліш** (Haiku, консервативний): чернетка → фінальний `.md` + `attentionNotes[]`.
   НЕ міняє зміст (цитати/суми/дати/імена) — §6 design. JSON depth-counter парсинг.
3. **Fallback**: немає layout (старий скан) → плоский `.txt` → AI. Немає API-ключа/AI кинув
   → зберегти чернетку (deterministic-only) + warning.

### Долі артефактів (Вісь A)
- `<basename>_<id>.md` — **створюється** у 02_ОБРОБЛЕНІ;
- `<basename>_<id>.txt` — **переміщається** у `02_ОБРОБЛЕНІ/_raw_txt/` (latin-safe ім'я, #8);
- `<basename>_<id>.layout.json` — **видаляється назавжди** (паливо відпрацювало);
- метадані: `textFormat:'md'`, `cleanedAt`, `attentionNotes`.

### schemaVersion 10 (рішення для 3.1 — structural, #6)
Поля `textFormat`(`'txt'|'md'`, default `'txt'`) і `cleanedAt` (nullable) у канонічну
`cases[].documents[]` = **bump schemaVersion 10** + `migrateToVersion10` (ідемпотентна,
default `textFormat='txt'` усім) + бекап `_backups/`. `attentionNotes` — **важке поле**,
у `documents_extended.json` (bump НЕ потребує).
> Один сенс (#11): `textFormat` — «у якому форматі лежить витягнутий текст». НЕ плутати
> з `documentNature` (природа файла) і `status` (lifecycle).
**Узгоджено з адвокатом ПЕРЕД 3.1** (structural — правило #6).

### C7-логування (Вісь A, дзеркало contextGenerator)
- agentType `textCleaner` у `SYSTEM_DEFAULTS` (`modelResolver.js`) → `'claude-haiku-4-5-20251001'`.
- `logAiUsage({agentType:'text_cleaner', context:{caseId, module, operation:'clean_text', documentId}})`
  + `activityTracker.report('agent_call', ...)` — один шлях на всіх споживачів.
- **Прапор `billAsUserAction`** (default true для кнопок/ACTION; DP передає false —
  автоматичне продовження обробки, як рішення contextGenerator). Один сенс (#11):
  «чи нараховувати виклик як окрему оплачувану дію адвоката». Токени в `ai_usage[]`
  пишуться завжди; `activityTracker` як дія — лише при true.

### СПІЛЬНІСТЬ ДИЗАЙНУ (наскрізне — для всіх UI-фаз 3.2/3.3/3.4)
Біль із минулого: CSS/стилі розповзлись по модулях, кожен тягне своє → морока зводити.
**Робити одразу спільним, не «потім рознесемо».** Для ВСЬОГО нового UI:
- **CSS / кольори / відступи / радіуси / тіні** — ТІЛЬКИ через наявні **design-токени**
  (CSS-перемінні проєкту). НЕ хардкодити hex/px по місцю, НЕ заводити нову палітру. Новий колір
  (напр. підсвітка `--attention-bg`) — **токен**, не літерал.
- **Шрифти** — наявні проєктні, без нових сімей.
- **Компоненти** — повторний елемент (чип, плашка, кнопка-дія, список-панель, компонент вибору) —
  **спільний компонент** у `components/shared/`, НЕ локальна копія в модулі.
- **Логіка/функції** — нуль дублювання: парсинг міток, стрип `==`, scroll-to-mark, механіка
  вибору файлів — спільні хелпери, переюзні між фазами і майбутнім (борг #47).
- **Перевірка перед комітом:** жодного нового `.css`-острова з власними літералами; усе веде до
  спільних токенів/компонентів. Бракує токена — додати **у спільне місце**, не локально.
> Правило-орієнтир (#11 + Rule of Three): стиль/елемент/функція у 2+ місцях — спільний з народження.
