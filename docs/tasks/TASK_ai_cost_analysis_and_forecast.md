4. **OCR через Google Document AI — НЕ в `ai_usage`** (це не Anthropic). Його
   вартість окремо (Google Cloud pricing). У звіті розділити: Anthropic-токени
   vs Document AI-сторінки — це дві різні статті витрат.

## КРОКИ

1. Витягнути `ai_usage[]` з УСІХ 8 джерел (актуальний + 7 бекапів, fileId вище)
   через Drive MCP, **злити в один масив і здедупити по `id`**. Файли ~5-7 МБ —
   якщо `read_file_content` обрізає, завантажити сирий JSON (`download_file_content`)
   і парсити. Зафіксувати у звіті: скільки унікальних записів усього, скільки додали
   бекапи понад актуальний (якщо 0 — підтвердження що LIFO нічого не з'їв).
2. Агрегації:
   - сума `totalTokens` і `estimatedCostUSD` за весь період;
   - розбивка по `model` (Haiku/Sonnet/Opus) — де основні гроші;
   - розбивка по `agentType` і `context.operation` — топ-дорогі операції;
   - розбивка по днях (`timestamp`) — динаміка;
   - розбивка по `context.caseId` / `module` — які справи/модулі найважчі.
3. Звірка: сума `estimatedCostUSD` ≈ реальне списання балансу (адвокат повідомить
   суму поповнення/залишок). Розбіжність пояснити (caching, неповне логування, OCR).
4. Прогноз: вартість «активного дня» → «активного місяця» (з поправкою на те, що
   тестування = ненормально інтенсивна нарізка; виділити «реальний робочий профіль»
   vs «стрес-тест»).
5. Рекомендація для SaaS: $X/місяць собівартості AI на активного користувача →
   вхід для тарифу і `subscription.limits` (зв'язок з боргом #31 — уніфікований
   стан «ліміт вичерпано»).

## МЕЖІ / НЕ ЧІПАТИ

- Це аналіз, не рефакторинг білінгу. НЕ міняти логіку `aiUsageService`/`activityTracker`.
- НЕ плутати `ai_usage[]` (токени, для оператора SaaS) з `time_entries[]` (час
  адвоката, для CRM) — дві окремі структури (CLAUDE.md).
- Розбіжність ліміту 50000/100000 — окремий борг #32, НЕ виправляти в цьому аналізі
  (лише відзначити вплив на повноту даних).
- НЕ робити повноцінний Billing UI (окремий TASK через 6+ міс).

## SAAS / BILLING IMPLICATIONS

Це фактично **перший реальний вхід у фінансову модель SaaS**: собівартість AI →
ціна пакета. Результат живить майбутній TASK Billing UI і налаштування
`subscription.limits`/`alerts` (warnAt:80, blockAt:100). Прив'язка до боргу #31
(уніфікований UX «ліміт вичерпано») і #22 (`entitlements`).
