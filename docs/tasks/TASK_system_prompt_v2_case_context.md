## Контекст

В Legal BMS у Досьє справи є функція **"Створити контекст"** яка викликає Claude API з документами справи і генерує файл `case_context.md` на Google Drive у папці справи.

Цей файл потім використовується **агентом досьє у чаті** як основний контекст для відповідей адвокату.

Поточний промпт для генерації цього файлу — **застарілий** і не покриває всіх випадків. Цей TASK замінює його на нову версію з:

- Адаптацією структури під тип справи (цивільна / адміністративна / кримінальна / господарська)
- Двосторонньою процесуальною позицією (для випадків зустрічного позову)
- Безсторонньою позицією з маркером клієнта в шапці файлу
- Адаптивним обсягом (16K як ліміт, не ціль)
- 8 жорсткими правилами антигалюцинацій
- Підтримкою експертиз з рецензіями
- Передачею нотаток адвоката для розділу спостережень

### Метадані

```javascript
function buildCaseMetadata(caseData) {
  return {
    CURRENT_DATE_ISO: new Date().toISOString().slice(0, 10),
    CASE_NAME: caseData.name,
    CASE_NUMBER: caseData.number,
    CASE_TYPE: caseData.type || '',
    CASE_CATEGORY: caseData.category || '',
    CURRENT_STAGE: caseData.current_stage || '',
    COURT_NAME: caseData.court || '',
    JUDGE_NAME: caseData.judge || '',
    OPENED_DATE_ISO: caseData.opened_date || '',
    CLIENT_NAME: caseData.client_name || '',
    CLIENT_ROLE: caseData.client_role || '',
    FIRST_INSTANCE_NUMBER: caseData.instances?.first?.case_number || '',
    APPEAL_INSTANCE_NUMBER: caseData.instances?.appeal?.case_number || '',
    CASSATION_INSTANCE_NUMBER: caseData.instances?.cassation?.case_number || '',
    DOCUMENTS_COUNT: caseData.documents?.length || 0
  };
}

function fillSystemPrompt(template, metadata) {
  let prompt = template;
  for (const [key, value] of Object.entries(metadata)) {
    prompt = prompt.replaceAll(`{{${key}}}`, value);
  }
  return prompt;
}
```

```javascript
  const {
    pinnedFirst = true,
    maxRecent = 5,
    maxPinned = 10
  } = options;
  
  // Знайти всі нотатки прив'язані до справи
  const caseNotes = (allNotes?.cases || [])
    .filter(n => n.caseId === caseId);
  
  // Закріплені нотатки (з cases[i].pinnedNoteIds)
  const pinnedIds = new Set(caseData.pinnedNoteIds || []);
  const pinned = caseNotes
    .filter(n => pinnedIds.has(n.id))
    .slice(0, maxPinned);
  
  // Останні незакріплені
  const recent = caseNotes
    .filter(n => !pinnedIds.has(n.id))
    .sort((a, b) => new Date(b.ts || b.createdAt) - new Date(a.ts || a.createdAt))
    .slice(0, maxRecent);
  
  return [...pinned, ...recent];
}

function formatNotesForPrompt(notes) {
  if (!notes || notes.length === 0) return '';
  
  const lines = ['НОТАТКИ АДВОКАТА (з Legal BMS):'];
  for (const note of notes) {
    const date = new Date(note.ts || note.createdAt).toISOString().slice(0, 10);
    const type = note.type || 'general';
    const isPinned = note.isPinned ? ' [закріплена]' : '';
    lines.push(`[${date}, ${type}${isPinned}]: ${note.text || note.content || ''}`);
  }
  return lines.join('\n');
}
```

```javascript
async function createCaseContext(caseData, allNotes, ocrTexts) {
  const metadata = buildCaseMetadata(caseData);
  const systemPromptFilled = fillSystemPrompt(SYSTEM_PROMPT_TEMPLATE, metadata);
  
  const notes = getNotesForContext(caseData.id, allNotes);
  const notesText = formatNotesForPrompt(notes);
  
  // Підготовка documents для messages
  const documents = ocrTexts.map((text, i) => ({
    type: 'document',
    source: {
      type: 'text',
      media_type: 'text/plain',
      data: text
    },
    title: `Документ ${i + 1}`,
    context: `Документ справи ${caseData.name}`
  }));
  
  const userMessage = notesText ? 
    `${notesText}\n\nПроаналізуй документи нижче і сформуй case_context.md.` :
    'Проаналізуй документи нижче і сформуй case_context.md.';
  
  const response = await callClaudeApi({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 16000,
    system: systemPromptFilled,
    messages: [
      {
        role: 'user',
        content: [
          { type: 'text', text: userMessage },
          ...documents
        ]
      }
    ]
  });
  
  return response;
}
```

Після впровадження протестувати на трьох справах різного типу:

1. **Цивільна з зустрічним позовом** (Брановський) — перевірити що структура 4.1/4.2/4.3 працює, нейтральність позицій збережена, маркер клієнта в шапці.

2. **Кримінальна** — перевірити що структура з 4.1/4.2/4.3/4.4 працює, потерпілий має повноцінний розділ незалежно від того кого представляємо.

3. **Мала справа** (тільки позов і ухвала про відкриття провадження) — перевірити що:
   - Файл компактний, не намагається заповнювати все 16K
   - Порожні розділи мають тільки заголовки без плейсхолдерів
   - Не з'являються вигадані факти

Перевірити що:
- Дати скрізь в ISO форматі
- Цитати дослівні з лапками і посиланнями
- Маркер клієнта тільки в шапці, не в розділах позицій
- Експертизи (якщо є) — окремий блок 7 з рецензіями як зноски
- Нотатки з закріплених проявляються в розділі 10
