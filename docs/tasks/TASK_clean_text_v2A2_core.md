## СКЛАДОВІ

### A2.1 — `mode` у ядрі + два промти + зберігання за суфіксом
- `cleanDocument({ ..., mode })`, `mode ∈ {'clean','digest'}`:
  - **`digest`** = поточний 3.1-промт (структурує/переказує = Конспект). Лишається як є.
  - **`clean`** = **НОВИЙ строгий промт** (Чистий). Залізні правила (урок Брановського):
    «прибери ТІЛЬКИ OCR-сміття (артефакти, дублі, розірвані переноси), віднови абзаци/заголовки.
    **НЕ переставляй, НЕ групуй, НЕ скорочуй, НЕ переказуй. НЕ міняй жодного слова, цифри, дати.
    НЕ міняй особу/рід/відмінок (не «я»→«позивач»). Поверни ТОЙ САМИЙ текст, лише без сміття.»
- `saveMarkdown` пише за суфіксом: `<base>_<id>.clean.md` або `.digest.md` (за `mode`).
  Старий `<base>_<id>.md` (з 3.1) — backward-compat: читається як digest (див. A2.6 міграція).

### A2.2 — долі артефактів: зберегти `.layout`, спростити
- **Прибрати `deleteLayout`** з success-шляху `cleanDocument` (3.1 його видаляв — скасовуємо; layout тепер джерело Точного й повторної генерації).
- Для нових сканів `.txt` не існує (A2.3) → `moveRawTxtToArchive` стає no-op/прибрати для них.
- Тобто на очистці: лише `saveMarkdown(.clean|.digest)` + `updateDocumentMeta(variants)`. Без руйнівних кроків.

### A2.3 — припинити писати `.txt` для Document-AI-сканів + фото-склейки
- У місцях OCR-пайплайну, що пишуть `.txt` (`writeProcessedArtifacts` у `splitDocumentsV3`; будь-який
  single-doc OCR write у CaseDossier) — **не писати `.txt`, коли є layout** (Document AI / image-merge).
- **Лишити `.txt`** там, де layout немає (pdfjsLocal малі скани, searchable/конвертер). Тобто `.txt`
  пишеться ⇔ layout відсутній.
- ⚠ Це той самий `splitDocumentsV3` — координація з DP-сесією (див. шапку).

### A2.4 — хелпер `getDocumentText(doc, caseData)` (єдина точка тексту)
- Консолідувати read-доступ: **scanned з layout** → текст із `page._text` (з'єднати сторінки, ≈ старий `.txt`);
  **no-layout/searchable/legacy** → читати `.txt`; (пізніше — текст-шар PDF через `pdfjsLocal`-бекенд, поки НЕ робимо).
- Будувати на базі `ocrService.getCachedLayout`/`getCachedText` (вже є). Повертає **вірний** текст (НЕ дайджест).
- Перевести споживачів на хелпер: `contextGenerator` (де брало `.txt`), агент (3.2 ACTION уже через ядро — звірити), будь-хто інший.
- **Агент/contextGenerator беруть вірне** (хелпер), **ніколи Конспект**.

### A2.5 — прибрати DP-пост-крок (повністю)
- Видалити з `splitDocumentsV3` гілку `cleanForReading`+`cleanFinalizedDocument` (додану в 3.1 `67fdda4`).
- Прибрати проводку тумблера «Очистити для читання» (`DocumentPipelineContext`, `DocumentProcessorV2`).
- DP більше **не запускає AI-очистку**. Ядро `cleanDocument` + adapter — **лишаються** (їх кличе в'ювер/3.2 ACTION).
- ⚠ `splitDocumentsV3` — координація з DP-сесією.

### A2.6 — schema bump 10→11 (`variants`)
- Канонічна схема `documentSchema.js`: додати `variants` — які AI-варіанти згенеровано, напр.
  `{ clean: <cleanedAt>|null, digest: <cleanedAt>|null }`. Один-реченнєвий сенс (#11).
- `migrateToVersion11`: усім документам `variants` дефолт; **backward-compat**: якщо `textFormat==='md'`
  (наявний 3.1-дайджест) → `variants.digest = cleanedAt`, інакше обидва null. Ідемпотентна, бекап PreV11.
- `updateDocumentMeta` оновлено писати `variants` (замість/поряд `textFormat`).
- Viewer «Текст»/`getCleanOrRawText` — читати за варіантами: digest (`.digest.md` або legacy `.md`) → інакше хелпер (вірний текст).

### A2.7 — 3.2 ACTION узгодити
- `clean_document_text` приймає `mode` (default — лишити поточну поведінку, тобто `digest`, щоб наявні
  виклики/кнопки не зламались; перемикач режимів додасть V2-B). Зберігає за суфіксом.
