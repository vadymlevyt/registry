## СКЛАДОВІ

### V2-B.1 — перемикач режимів (за `documentNature` + layout + `variants`)
- **scanned:** `Скан / Точний / Чистий / Конспект`. Точний — активний коли є layout (як V2-A1).
  Чистий/Конспект — таб завжди видимий, але стан: **готово** (`variants[mode]` є) → активний таб;
  **не згенеровано** → таб із підписом «Згенерувати ✨».
- **searchable:** `Документ / Конспект`. Точний/Чистий **не показуються** (нема OCR-сміття/layout).
  Конспект — універсальний (parent).
- **Прибрати таб «Текст»** — його поведінки розкладені: дайджест → таб Конспект; сире/механічне → Точний.
- **Дефолт:** scanned → **Точний** (миттєвий, faithful); searchable → **Документ**.
- ✨-маркер на Чистий/Конспект (AI). Badge «⚠ переказ, не дослівно» — на Конспекті.

### V2-B.2 — генерація на вимогу
- Клік по незгенерованому Чистий/Конспект → підтвердження «Згенерувати (AI, ~N хв)?» → `executeAction(agentId,
  'clean_document_text', { caseId, documentId, mode })`.
- Поки генерується — **прогрес** у тілі таба (спінер + «Очищаю пачку i/N», якщо `onProgress` прокинуто;
  інакше просто спінер «Очищаю...»). Помилка/деградація (`ok:false`) → toast, таб лишається «Згенерувати».
- На успіх — `document.variants[mode]` оновився → таб стає активним, показуємо `.md`.

### V2-B.3 — рендер готових варіантів
- Чистий → читати `<base>_<id>.clean.md`; Конспект → `.digest.md` (+ legacy `<base>_<id>.md` як digest).
- Рендер через `MarkdownRenderer`. (Читання — через наявні `ocrService`-аксесори за режимом; helper-родина.)

### V2-B.4 — searchable-Конспект
- Для searchable вхід генерації — текст через `getDocumentText` (хелпер, V2-A2) → ядро `cleanDocument(mode:'digest')`.
  (Скоуп-гард ядра — лише scanned для clean; для digest допустити searchable: звірити/розширити гард.)
  ⚠ Якщо ядро `cleanDocument` має скоуп-гард `scanned-only` — для searchable-Конспекту його треба пом'якшити
  (Конспект універсальний). Це невелика правка ядра — узгодити; решта clean (Чистий) лишається scanned-only.
