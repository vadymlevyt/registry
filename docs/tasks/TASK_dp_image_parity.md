## МЕТА

DP-редактор склейки фото (`DpImageMergeEditor`, TASK 1B) **не підтягнув** з модалки два
готові контролі, і має UI-баг накладання:
- **#1** не показує **дублі** (модалка бере в жовту рамку, зеленим — рекомендований, дає вибрати);
- **#9** не дає **контролю обрізки** (модалка: банер «Не обрізати жодну» + per-thumb ✕; DP сам
  обрізає без глобального керування);
- **#4** при відкритті попапа (збільшеного фото) **меню досьє налазить поверх** попапа.

Усі три — **зона image-merge editor**. Контролі #1/#9 **уже існують у модалці** (`PreviewView`
+ спільні `ImageEditor/*`) — їх треба **перенести у DP зі спільного місця**, не дублювати
(правило «Спільний рендер UI»). #4 — z-index/portal у спільному `PreviewPopup`.

> Зона «Контекст» (#7/#5/#2/#3/#6) — **інша сесія**, не чіпати.

## СКЛАДОВІ

### #1 — Дублі в DP (паритет з модалкою)
**Файли:** `DocumentProcessorV2/DpImageMergeEditor.jsx` (+ можливо тонкий винос спільної
duplicate-логіки, якщо вона зараз інлайн у модалці `ImageMergePanel/index.jsx`).
- Звідки дані: `sortResult` від `imageSortingAgent` (містить інфо про дублі), що вже рахується
  на етапі `prepareImagesForMerge` у DP image-flow. Прокинути дублі у `DpImageMergeEditor`.
- Обчислити `duplicateMembership` (як модалка) і передати у спільний grid → `Thumbnail`/`RenderItem`
  малюють **жовту рамку** на дублях, **зелений** на рекомендованому (вони це вже вміють).
- Рендерити `duplicateInfo` блок («Знайдено N груп дублікатів…») і дати вибір/зняти —
  **перенести зі спільного** (якщо логіка інлайн у модалці — винести у спільне місце, щоб
  обидва споживачі тягнули; правило #30).

### #9 — Контроль обрізки в DP (паритет)
**Файли:** `DpImageMergeEditor.jsx`.
- Додати **crop-банер** як у `PreviewView:310-323`: «Обрізку буде застосовано до N стор…
  **Не обрізати жодну**» (`onSkipAllCrops`) — коли є cropProposals. Реалізувати `onSkipAllCrops`
  (зняти всі) і, якщо модалка має, `onApplyCropToAll`.
- Per-thumb ✕ вже є (`onToggleCropDisabled`) — переконатись що видимий і працює як у модалці.
- **Звірити дефолт із модалкою:** обрізка за AI-пропозицією застосовується за дефолтом, але
  адвокат може зняти (per-thumb або «Не обрізати жодну»). Поведінка фінальної збірки
  (`rebuildFromOcrResults`) має точно збігатися з модалкою — однакова обрізка → однаковий PDF.

### #4 — Overlay: попап поверх меню досьє
**Файли:** `ImageEditor/PreviewPopup.jsx` (+ можливо CSS).
- Зробити так щоб `PreviewPopup` рендерився **поверх** chrome досьє (агент-панель/меню) у
  DP-контексті: портал у корінь документа + достатній z-index. 
- **Не зламати модалку** — у модалковому контексті попап має лишитись як є. Спільний фікс
  (portal/z-index) має працювати в обох.
- Пов'язано з боргом #26 (CSS-префікс `image-merge-panel__` у спільному) — якщо торкаєшся
  CSS попапа, можна (опційно) почати міграцію префікса; якщо ні — лишити, не розширювати scope.
