## ЧОМУ v2 (корінь)

Тест на реальній позовній заяві (19 стор., Брановський) показав: однорежимна очистка
Хайку дає **гарний, але НЕВІРНИЙ** результат — AI **переказав і переструктурував**
(згрупував статті, виніс цитати), **скоротив** дослівні цитати ВС, **змінив особу і рід**
(«Я… не користувалась» → «Позивач… не користувався»). Тверді дані (числа/дати/імена) — вірні,
але **текст уже не дослівний**.

Для юрдокумента це фатально: інколи треба **точну цитату** з рішення скопіювати. Водночас
адвокату **структурований конспект зручніший** для швидкого розуміння (юристи пишуть важко).

**Висновок:** не один режим «зробити гарно», а **три** — розділити **ВІРНЕ** і **ЧИТАБЕЛЬНЕ**.

## ТРИ РЕЖИМИ

| Режим | Що це | AI | Вірність | Скоуп | Швидкість |
|-------|-------|----|---------|----|-----------|
| **Точний** | механічне вирівнювання абзаців/заголовків (КРОК 1, `layoutToMarkdownDraft`) | ❌ 0 токенів | **100% дослівно** (за побудовою) | тільки scanned | миттєво, безкоштовно |
| **Чистий** | AI прибирає ТІЛЬКИ OCR-сміття, нічого не переставляє/не міняє | ✅ строгий промт | вірний за наміром (**треба тести**) | тільки scanned | повільно (AI) |
| **Конспект** | AI структурує/переказує для читання (поточний 3.1-промт = режим 3) | ✅ | **НЕ дослівно** (переказ) | **scanned + searchable** | повільно (AI) |

**Контракт вірності (критично):**
- **Точний** — гарантовано дослівний (детермінований, не вміє міняти слова). Базовий для цитат.
- **Чистий** — намір «лише де-шум», але AI може дрейфувати → **підтвердити тестами** перед довірою.
- **Конспект** — **явно позначений «⚠ переказ, не дослівно»**. Лише для людського швидкого читання.
  **НІКОЛИ** не джерело для агента/цитат.

> Точний/Чистий — про OCR-сміття (тому тільки scanned; searchable і так чистий). **Конспект —
> універсальний**: гарно написаний документ опонента/прокурора теж варто стиснути для читання.

## ДЖЕРЕЛО ТЕКСТУ — хелпер `getDocumentText` (#11, єдина точка)

Єдиний read-аксесор «дай вірний текст документа», що абстрагує **де** текст лежить:
- скан → з `.layout.json` (`page._text`);
- searchable/legacy → з `.txt` (поки що);
- (потім, коли приберемо `.txt` для searchable) → текстовий шар PDF через `pdfjsLocal`-бекенд.

**Будуємо як еволюцію** наявної родини `ocrService.getCleanOrRawText`/`getCachedText`/`getCachedLayout`
(read-аксесори, вже існують). `pdfjsLocal` — **НЕ зливаємо** (це інгест-екстрактор, інший шар);
хелпер **делегує** йому екстракцію для searchable.

Споживачі (в'ювер, агент досьє, генерація конспекту, contextGenerator) ходять **через хелпер** →
коли пізніше приберемо `.txt`, міняється тільки хелпер. Агент завжди бере **вірне** джерело (layout/Точний), **не Конспект**.

## ДОЛЯ `.txt` І ЗБЕРІГАННЯ

**`.txt`:**
- **Document-AI скани + фото-склейка** (layout є завжди — pdfjsLocal не OCR-ить картинки, тільки Document AI) → **перестаємо писати `.txt` ВЖЕ ЗАРАЗ** (layout містить текст).
- **searchable** (pdfjsLocal/конвертер) і **legacy** (наявні `.txt` без layout) → `.txt` лишається; хелпер читає; повне прибирання — окремий пізніший рефакторинг.

**Зберігання — у `02_ОБРОБЛЕНІ`, за суфіксом імені (без нових папок):**

| Файл | Що |
|------|-----|
| `<base>_<id>.layout.json` | сире джерело (скан) — **зберігаємо, НЕ видаляємо** (раніше видаляли — скасовано) |
| `<base>_<id>.txt` | сирий текст (лише searchable/legacy) |
| `<base>_<id>.clean.md` | Чистий — якщо згенеровано |
| `<base>_<id>.digest.md` | Конспект — якщо згенеровано |
| *(Точний)* | **не зберігаємо** — рахуємо на льоту з layout |

**Схема:** документ трекає, які варіанти існують (`variants`) → **bump schemaVersion** + ідемпотентна міграція + бекап.

## DP БІЛЬШЕ НЕ ЧИСТИТЬ

Прибираємо DP-пост-крок очистки (доданий у 3.1: `cleanFinalizedDocument` у `splitDocumentsV3`).
Причини: Точний рахується на льоту (нема що робити в пайплайні); Конспект/Чистий на весь том =
години (недоречно як авто-крок). Очистка стає **справою в'ювера** (по одному документу, на вимогу).
Бонус: зникає проблема «DP молотить том 5+ хв і деградує».

Ядро (`cleanDocument`) і adapter — **лишаються** (їх тягне в'ювер). Прибираємо лише DP-проводку.

## ІНТЕРФЕЙС В'ЮВЕРА (деталі — V2-B)

Перемикач представлень (розширює наявний Скан/Текст):
```
скан:       [ Скан ] [ Точний ] [ Чистий ✨ ] [ Конспект ✨ ]
searchable: [ Документ ] [ Конспект ✨ ]              ✨ = AI
```
- Точний — дефолтний показ тексту скана (live).
- Чистий/Конспект — якщо не згенеровано: таб неактивний з «Згенерувати (AI, ~N хв)»; клік → генерує+зберігає+показує. Далі — миттєве перемикання.
- **Стрімінг**: AI-режими течуть у в'ювер по мірі генерації (як claude.ai) — прибирає відчуття зависання.
- **Конспект** несе badge «⚠ переказ, не дослівно».
- Підсвітки уваги (колишня 3.4) — лише на AI-режимах.
