## СКЛАДОВІ

### 3.3.1 — Спільний компонент вибору
- Винести у `components/CaseDossier/FileSelectionList` (або `components/shared/SelectableList`):
  рендер списку з чекбоксами, select-all (indeterminate), батч-панель дій.
- Дії — через **props-слоти** (різні в Архіві vs реєстрі): масив `{label, variant, icon, onRun}`.
- Стан `selectedIds`(Set) — у батьку (як зараз `selectedArchivedIds`), компонент керований.
- **CSS уніфікувати** — спільний клас, один шлях (не дублювати `archive-*`).

### 3.3.2 — Архів на спільний компонент (behavior-preserving)
- `ArchiveView` рефакторити: рендер списку+вибір — через спільний компонент; дії
  (відновити/видалити обрані/всі) лишаються через props-слоти.
- **Поведінка ІДЕНТИЧНА** (тести Архіву зелені без змін очікувань).

### 3.3.3 — Реєстр: режим вибору + дві батч-дії
- У реєстрі матеріалів (дерево/список) — режим вибору (чекбокси, select-all) через спільний компонент.
- Не ламати поточний клік→Viewer (режим вибору — окремий стан/перемикач).
- Батч-панель: **«Очистити вибрані»** + **«Видалити вибрані»**.

### 3.3.4 — «Очистити вибрані» (тягне ядро 3.1)
- Для вибраних: фільтр scanned (parent скоуп); для вже-`.md` — перепит «Перезапустити очистку?»
  (так→reclean, ні→skip). Цикл `cleanDocument`, прогрес, підсумок.

### 3.3.5 — «Видалити вибрані» (пакетне, повне)
- Підтвердження (батч-діалог із кількістю).
- Для кожного **повне видалення** (переюзати наявне):
  - файл у `01_ОРИГІНАЛИ` (`deleteDriveFile`);
  - `.txt`+`.layout.json` (`deleteOcrCacheForDocument`);
  - **`.md`** якщо очищений (РОЗШИРИТИ `deleteOcrCacheForDocument`: додати `.md`-ім'я; +
    `_raw_txt/` копія якщо є);
  - метадані з реєстру (`delete_document` ACTION).
- Підсумок (видалено N, помилок K).
