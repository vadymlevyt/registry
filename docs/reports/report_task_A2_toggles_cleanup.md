## 1. ПОЯСНЕННЯ ДЛЯ ВЛАСНИКА (проста мова)

У панелі налаштувань Document Processor висіло дев'ять перемикачів. Виявилось, що **п'ять із них
нічого не робили** (намальовані, але логіки за ними нема), і **три з цих порожніх стояли
увімкненими** — тобто галочка є, а за нею порожнеча. Це був обман інтерфейсу.

Зроблено:
1. **Викинули п'ять мертвих перемикачів** (цілісність, дедлайни, короткий зміст, картка справи,
   розкласти по провадженнях). Backing-логіки під ними не було жодної — прибрали лише UI.
2. **Перевернули замовчування:** тепер за умовчанням «просто додати файли», а обробка — свідомий
   вибір. Один перемикач **«Нарізати / склеїти»**: коли увімкнено, система **сама** обирає за
   типом входу — PDF-том нарізає на документи, фото склеює в документ; коли вимкнено (за
   умовчанням) — просто додає файли як є.
3. **Перемикач збору прикладів для власної моделі нарізки** сховали так, що його бачить **тільки
   власник**.

**Результат:** у панелі лишились тільки робочі перемикачі, нічого зайвого, обробка вмикається
свідомо. Жодного коду «під капотом» не зламано — мертві перемикачі ні до чого не були підключені.

## 3. ФІНАЛЬНА МОДЕЛЬ ТУМБЛЕРА (для тесту руками)

Один тумблер **«Нарізати / склеїти»** (за умовчанням ВИМКНЕНО):
- **OFF (дефолт):** просто додати файли як є (будь-який тип).
- **ON + фото** → редактор склейки.
- **ON + сканований PDF-том** → нарізка на документи.
- **ON + Word / малий-цифровий PDF** → ввічливий toast «лише сканований PDF».
- **ON + мікс фото+PDF** → toast «вимкніть Нарізати / склеїти».
- **ZIP (ЄСІТС)** → розпак → просто додати (підписи `.p7s` відкидаються).
