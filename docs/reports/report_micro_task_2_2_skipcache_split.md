# Звіт мікро-TASK 2.2 — розділити skipCache на читання і запис

**Дата:** 2026-05-09
**Виконавець:** Claude Code (Opus 4.7, 1M context)
**Статус:** Завершено. 422/422 тестів зелені.
**Передумова:** Швидка діагностика підтвердила що `writeCache` не викликається при reprocess через спільний прапор `skipCache`.

---

## Що зроблено

З умови запису кеша в `ocrService.extractText` видалено `&& !options.skipCache`. Тепер прапор `options.skipCache` керує **лише читанням** старого кеша (рядок 174 — обхід `checkCache` коли адвокат натискає «Розпізнати зараз»). **Запис** свіжого результату відбувається завжди, коли провайдер повернув непорожній текст: `if (result.text && result.text.trim().length > 0)`. Це логічно правильна поведінка — при перерозпізнаванні старий кеш-файл у `02_ОБРОБЛЕНІ` буде перезаписаний новим (`writeCache` робить це через цикл `existing.forEach(DELETE)` перед `uploadTextFile`). Обрано **мінімальний варіант** замість окремих прапорів `skipCacheRead` / `skipCacheWrite` — менше поверхні для регресії, поточний єдиний caller (`onReprocess` у CaseDossier:2450) і батч-обробка (DocumentProcessor) обидва хочуть писати кеш, тому розрізнення не потрібне зараз. Доданий коментар-пояснення.

## Як перевірити

### Тест — Кісельовин документ «Ухвала»

1. Відкрий справу Кісельова → документ «Ухвала».
2. Натисни **«Розпізнати зараз»**.
3. **Очікувано:** упродовж 1-3 секунд з'являється текст; справа знизу **зелений** toast «Текст розпізнано і збережено» (не помаранчевий warning).
4. Закрий і відкрий той самий документ → текст з'являється **миттєво** (з кешу, через `getCachedText`).
5. Перевір на Google Drive у папці справи `02_ОБРОБЛЕНІ` — має бути файл `<sanitized_name>_<driveId>.txt`. Відкрий його — побачиш OCR-текст.

### Опційне підтвердження через DevTools

1. Перед натисканням «Розпізнати зараз» очисти Network tab.
2. Натисни кнопку, дочекайся toast.
3. У Network має бути запит **`POST upload/drive/v3/files?uploadType=multipart`** з status 200 — це і є запис кеша. Тіло відповіді містить `id` і `name` створеного файлу.
4. Якщо запит є і status 200 — все зроблено правильно. Якщо запит є але status 4xx — то вже окрема проблема Drive permissions (поза скопом цього мікро-TASK).

### Що змінилось у поведінці порівняно з мікро-TASK 2.1

- **Раніше:** натискання «Розпізнати зараз» → OCR відпрацьовує → текст показується → але `writeCache` не викликається (бо `skipCache=true` блокувало і запис) → жовтий warning toast → повторне відкриття документа знов показує empty state.
- **Зараз:** натискання «Розпізнати зараз» → OCR → `writeCache` пише файл у `02_ОБРОБЛЕНІ` → зелений success toast → повторне відкриття миттєво показує текст з кеша.
