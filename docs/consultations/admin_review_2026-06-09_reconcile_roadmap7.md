## 2. УТОЧНЕННЯ, яких §7 ще не вирішив (net-new від сесії)

### 2.1 «Один важіль» — це UI-пресет над ДВОМА полями, не одне поле (правило #11)
§7.3 формулює: «**Один важіль** зв'язує §7.2+§7.3: керує OCR→лейаут / вижимка / нічого + рівнем у контексті». Це **два різні питання на одному імені**:
- **(a) стан розпізнавання** — чи створено персистентний артефакт у `02` (керується `ocrMode`, зараз runtime/інференс);
- **(b) рівень участі в контексті** — Повний / Вижимка / Не включати.
Документ може бути розпізнаний (`full`), але **не включений** у контекст — і навпаки (вижимка не потребує повного лейаута). Зливати в одне поле = кейс `skipCache` (тонкий баг, проходить тести).
**Рекомендація:** на рівні даних — **два ортогональні значення**; «важіль» — **UI-пресет**, що пише обидва (пресети адвокатською мовою: «Повністю» / «Лише згадка» / «Не аналізувати»). Мапінг пресет→2 значення задокументувати на місці оголошення.

### 2.2 Асиметрія персистентності: `ocrMode` (runtime) vs participation (потребує поля схеми)
`ocrMode` свідомо НЕ в схемі (інференс). Але **рівень участі в контексті** інференсом не відновиш — це рішення адвоката, яке має пережити перезавантаження → **персистентне поле документа** (likely bump v12 + ідемпотентна міграція, default «Повний», бекап; правило #6). Це ще одна причина, чому (a) і (b) — різні сутності (одна інфериться, інша зберігається).

### 2.3 Звірити «Повний» з наявним `isKey` — не плодити поле
`isKey` уже існує і за нотаткою метадані→граф несе сенс «участь у контексті». Перед введенням нового поля — перевірити, чи «Повний/ключовий» рівень не є тим самим `isKey` (правило #11: розширити існуюче, не дублювати). Можливо, нове поле — це лише `Вижимка|Не включати` поверх булевого `isKey`, або `isKey` поглинається новим enum. Рішення — на старті спека §7.3.

### 2.4 DOCX/HTML `.txt` (едж §7.1) — зберігаємо
Рекомендація: `.txt` для DOCX/HTML — **легітимне єдине сховище** (конвертований PDF може бути растровим; повторний OCR втратний/платний). Не синтезувати layout (YAGNI). Зафіксувати в CLAUDE.md як норму для `documentNature='searchable'` без layout.
