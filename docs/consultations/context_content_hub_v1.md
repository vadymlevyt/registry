## КЛЮЧОВІ СЦЕНАРІЇ

### Сценарій 1 — Дайджест (автоматичний, щопонеділка)
```
n8n 07:00 → збирає джерела →
Opus аналізує через три фільтри →
Результат в Legal BMS (розділ Дайджест) →
Адвокат: [Контент] / [Ідея] / [Ігнорувати] →
Паралельно: Sonnet готує зовнішній дайджест для Telegram
```

### Сценарій 2 — Термінове сповіщення
```
n8n тригер (нова публікація ВС або зміна закону) →
Opus: три фільтри (релевантність → новизна → вікно) →
Якщо пройшло всі три → Telegram адвокату:
  «⚡ ТЕРМІНОВА ТЕМА
   Що: [суть]
   Чому важливо тобі: [паралель зі справами]
   Вікно: [скільки годин тому, чи є публікації]
   Оцінка: [висока/середня/низька]
   [Відкрити] [Нагадати завтра] [Ігнорувати]»
```

### Сценарій 3 — Генерація тексту
```
Вхід з трьох місць:
  — Дайджест → [Контент]
  — Банк ідей → [Генерувати]
  — Вручну в формі генерації

Sonnet генерує 5 варіантів + Opus для Stories →
Матриця публікацій (хто куди йде) →
Редактор і схвалення →
[Запланувати] → n8n публікує в час X
```

### Сценарій 4 — Відео з адвокатом
```
Адвокат знімає → скидає в Drive/Raw/ →
n8n бачить новий файл →
Descript + Underlord:
  транскрипція → filler words → паузи → субтитри → 9:16 →
Плеєр в Legal BMS →
Адвокат переглядає → [Схвалити] / [Повернути] / [Відхилити] →
Sonnet генерує підписи під кожну платформу →
Матриця публікацій → планувальник → n8n публікує
```

### Сценарій 5 — Відео без адвоката
```
Адвокат пише тему або обирає з банку ідей →
Opus генерує сценарій (гук 5 сек + суть 40 сек + заклик 10 сек) →
Адвокат схвалює сценарій →
Паралельно:
  ElevenLabs: текст → аудіо (клонований голос)
  Nano Banana: описи сцен → відеоряд
Збирається фінальне відео →
Плеєр → перегляд → схвалення → публікація
```

### Сценарій 6 — Idea Capture з будь-якого модуля
```
Адвокат в будь-якому місці системи →
Кнопка [💡 Ідея для контенту] →
Міні-модалка: текст ідеї + тип + контекст справи →
Зберегти → потрапляє в банк ідей зі статусом «нова»
```

## МАТРИЦЯ ПУБЛІКАЦІЙ — ЛОГІКА

Захист від накладок: не можна два варіанти однієї теми
на одну платформу.

```
ТЕМА: [назва]

Варіант          TG    IG    FB    TK    YT
─────────────────────────────────────────
TG-версія        [✓]   [ ]   [ ]   [ ]   [ ]
IG-карусель      [ ]   [✓]   [ ]   [ ]   [ ]
FB-текст         [ ]   [ ]   [✓]   [ ]   [ ]
YT-опис          [ ]   [ ]   [ ]   [ ]   [✓]
─────────────────────────────────────────
[Один варіант скрізь ↓]
```

«Один варіант скрізь» — окрема кнопка. Вибираєш найкращий
варіант і він адаптується технічно під всі обрані платформи
(хештеги, довжина) але текст той самий.

Захист: якщо намагаєшся поставити дві галочки в одному
стовпці → попередження «Telegram вже отримає TG-версію. Замінити?»

## ГОЛОВНИЙ АГЕНТ — ЗВ'ЯЗОК З CONTENT HUB

Content Hub — один зі стільників. Головний агент знає про нього
але не втручається в його роботу. Взаємодія через спільний стан:

ideas[] — банк ідей (читає і Content Hub і головний агент)
contentDecisions[] — рішення адвоката по контенту (логується)

Головний агент використовує contentDecisions[] для:
- Тижневого аналізу що заходило аудиторії
- Оновлення профілю вподобань
- Рекомендацій по темах виходячи з практики бюро
