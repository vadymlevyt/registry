# Ручний тест — додавання файлів (add-files + ZIP + повне комбо)

**Тип:** ручна QA-карта (жива; орієнтир коли щось десь вилізе)
**Дата:** 2026-06-09
**Скоуп:** сценарій «просто додати» (модалка + Document Processor), ZIP-інгест ЄСІТС,
повне комбо. Регресія нарізки/склейки — теж тут.

**Як користуватись:** іди по пунктах, кожен = дія → очікуваний результат. Де не так —
зафіксуй № пункту + що бачив. **На що дивитись усюди:** файл реально у `01_ОРИГІНАЛИ`,
відкривається у в'ювері, тости людські, нема «Failed to fetch».

---

## РОЗДІЛ 1 — Add-files без ZIP  ✅ протестовано 2026-06-08 (базово ок)

### 1A. Модалка «+ Додати документ» — з ПРИСТРОЮ
Для кожного: додати → «Додати документ» → тост «Документ додано», файл у матеріалах і в `01_ОРИГІНАЛИ`, видно у в'ювері.
1. Текстовий PDF → видно текст.
2. Сканований PDF (тумблери OFF) → «Текст розпізнано» (layout у `02`).
3. Сканований PDF + «Без розпізнавання» → швидко, без розпізнавання, видно.
4. Сканований PDF + «Стиснути» → тост-лічильник «Стиснення: X/Y», менший розмір у `01`.
5. HEIC / фото → не сіріє у пікері, → PDF, видно.
6. DOCX → PDF, видно.
7. `.doc` (старий) → додається, у в'ювері прев'ю через Drive (Word).
8. `.doc`/HEIC через системний вибір Android, що тягне з Drive → додається (НЕ «Failed to fetch»).

### 1B. Модалка — Drive-пікер («З Google Drive»)
9. Файл з Drive → завантажується → **копія в `01_ОРИГІНАЛИ`** (не посилання), конвертується/стискається як з пристрою, видно.

### 1C. Document Processor — «Просто додати»
10. Комбо (текстовий PDF + скан + DOCX + HTML + фото) + «Просто додати» → усі додані, кожен своїм шляхом, **без екрана «блок X з N»**.
11. «Просто додати» + «Без розпізнавання» → у `01`, без артефактів `02`.
12. «Просто додати» + «Стиснути» → стиснулись скани/фото, тост-прогрес.
13. Багатосторінковий PDF + «Просто додати» → **один документ, усі сторінки** (не порізаний).
14. Після прогону — **тумблери скинулись**.

### 1D. Нарізка / склейка — регресія (НЕ має зламатись)
15. Сканований багатодокументний PDF БЕЗ «Просто додати» → нарізка (екран «блок X з N»), ріже на документи.
16. Усі зображення, тумблер OFF → редактор склейки фото.

### 1E. Великий том
17. ~200 МБ скан + «Стиснути» + «Без розпізнавання» → видно тост-прогрес стиснення, завершується, додається.

## РОЗДІЛ 2 — ZIP-інгест (Document Processor)  ⬜ тестувати

> ЄСІТС віддає ZIP: документи (часто searchable PDF / HTML), додатки (скани, фото) +
> файли підписів `.p7s`/`.sig`. Зразки: `test_files/esits_samples/`.
> Сценарій: завжди **через Document Processor + тумблер «Просто додати»** (архіви в модалці НЕ приймаються).

18. **ЄСІТС-ZIP (мішаний вміст)** + «Просто додати» → архів розпаковано; **усі документи з нього додані** кожен своїм шляхом (PDF як є, HTML→PDF, фото→PDF, скан розпізнано); **файли `.p7s`/`.sig` відкинуто**; тост на кшталт «Розпаковано N файлів; M підписів відкинуто».
19. **ZIP, де є лише 1 документ + кілька підписів** → доданий 1 документ, підписи відкинуто (не з'являються у матеріалах).
20. **ZIP з вкладеними папками** → файли витягнуто (по іменах), папки проігноровано, дублів немає.
21. **ZIP + «Без розпізнавання»** → вміст доданий без розпізнавання (лише `01`, без `02`).
22. **ZIP + «Стиснути»** → скановані PDF/фото з архіву стиснулись (текстові — як є).
23. **Кілька ZIP одночасно** → усі розпаковані, вміст злитий, усе додано.
24. **Кирилиця в іменах файлів усередині ZIP** → назви збереглись коректно (не «крякозябри»).
25. **RAR / 7z** → НЕ розпаковується, додається як один файл + повідомлення «не розпаковується».
26. **Порожній ZIP / ZIP лише з підписами** → нічого не падає; зрозуміле повідомлення «немає файлів для додавання».
27. **Пошкоджений ZIP** → не валить застосунок; повідомлення про помилку, решта файлів (якщо є) додаються.
28. **Архів (.zip) у МОДАЛЦІ** → підказка «Архіви додавайте через Document Processor», ZIP як файл **не** додається.
29. **Великий ZIP** (десятки МБ / багато файлів) → розпаковується за можливостями пристрою; стежити, чи не «висне» без сигналу (якщо так — записати).

## РОЗДІЛ 3 — ПОВНЕ КОМБО (ZIP + усе інше разом)  ⬜ тестувати

> Мета: переконатись, що ZIP-розпакування і звичайні файли йдуть **однією трубою**,
> кожен своїм шляхом, тумблери діють на ВСЕ однаково.

30. **DP: 1 ЄСІТС-ZIP + окремий текстовий PDF + окреме фото + окремий DOCX** разом, тумблер «Просто додати» → 
    - вміст ZIP розпаковано (підписи відкинуто), 
    - окремі файли додані як є/конвертовані, 
    - **усе разом** у матеріалах і в `01_ОРИГІНАЛИ`, кожен відкривається; 
    - **без екрана нарізки**.
31. **Перевірка кількості:** порахуй документи у ZIP (мінус підписи) + окремі файли = стільки нових документів у справі. Нічого не загубилось, нічого зайвого (підписів нема).
32. **Те саме + «Стиснути»** → стиснулись лише скани/фото (з архіву і окремі); текстові PDF/DOCX/HTML — без стиснення.
33. **Те саме + «Без розпізнавання»** → усе (з архіву і окреме) додано без розпізнавання, лише `01`.
34. **2 ZIP + кілька окремих файлів** → усі архіви розпаковані, усе злито в один набір, усе додано.
35. **Після комбо-прогону** → тумблери скинулись; справа містить усі очікувані документи; жоден не «зламаний» у в'ювері.
36. **Мікс із RAR серед набору** (1 zip + 1 rar + окремі) → zip розпаковано, rar доданий як є + повідомлення, окремі додані; усе разом.

## Підсумок очікувань (швидка пам'ятка)
- ZIP → **розпаковується**; `.p7s`/`.sig` → **геть**; вміст + окремі файли → **addFiles, кожен своїм шляхом**.
- Тумблери «Без розпізнавання» / «Стиснути» діють на **весь** результат однаково; після прогону **скидаються**.
- Архіви — **тільки DP**; у модалці — підказка.
- RAR/7z — **як є** + повідомлення. Нарізка/склейка — **не зачеплені**.
- Усе лягає в `01_ОРИГІНАЛИ` і відкривається у в'ювері.
