## 4. Що з reuse — слідуємо Rule of Three

З їхнього файлу (правильно):

```
ImageMergePanel компоненти зараз — у CaseDossier/ImageMergePanel/.
Доки тільки CaseDossier їх використовує — це нормально. Один споживач.

Коли DP-Фаза 3 додасть image-merge UI — це ДРУГИЙ споживач.
За Rule of Three — поки не виносити, копіювати + tracking_debt.

Коли з'явиться ТРЕТІЙ (наприклад, окремий "Сканер документів" модуль
для адвоката-помічника) — тоді виносимо у components/ImageEditor/.
```

**АЛЕ** — у нашому випадку DP-Фаза 3 НЕ копіює, а **імпортує**. Це працює:
```js
import { rebuildFromOcrResults } from '../CaseDossier/ImageMergePanel/pdfRebuild';
import { PreviewPopup } from '../CaseDossier/ImageMergePanel/PreviewPopup';
```

Імпорт з нутрощів сусіднього модуля — **некрасиво** (нар. coupling), але працює і **не дублює** код. Це **тимчасова міра** до моменту коли з'явиться третій споживач або стане очевидно що інтерфейс stable.

**Тригер виносу у спільне:** третій споживач АБО **очевидна стабільність API** після DP-Фази 3.

**DrivePicker — це інший випадок.** Там РЕАЛЬНА дублікація (два файли, два роди коду). Виносимо при AddModal split (тиждень 8-9).

## 6. Одне зауваження від мене особисто

Адвокат-чат сесія дала **чудовий аналіз** reuse-патернів. Конкретно — частина 2 («Канонічні патерни code reuse») — це **майже доктринально правильно**:

- DRY, Single Source of Truth, Layered Architecture, Service Layer pattern — все правильно процитовано.
- Аналогія «не ієрархія модулів, а рівноправні брати-споживачі» — критично важлива.
- Rule of Three — головний практичний інструмент проти **передчасної абстракції** (overengineering).
- Чесний аудит реального коду з ✅/⚠️ — без вигадок.

Я з усім цим згоден. Адвокат-чат сесія додатково підтверджує **здоровий організм** проекту: сервісний шар вже зроблено правильно (converterService, ocrService, driveService, documentFactory — всі shared), фічові модулі — рівноправні споживачі. Це означає що **архітектурна підвалина міцна** — нам не треба переробляти фундамент перш ніж рухатись далі. Лише природний винос (Rule of Three) коли тригер активується.
