# _harvest_inbox — тимчасова staging-зона для жнив знань

**Призначення:** сюди власник кладе ZIP-архіви експортів зі сторонніх Claude-чатів
(старі роадмапи, ідеї, контекст). Окрема сесія-жнив їх опрацьовує разом із репо-файлами
(див. `docs/consultations/handoff_2026-06-27_knowledge_harvest_kickoff.md`).
**Папка ТИМЧАСОВА** — після жнив (коли зерно вкладено в канонічні доки) видаляється.

## Для власника
Поклади сюди `.zip` файли **як є — НЕ розпаковуй вручну**. Закоміть. Усе.

## Для сесії-жнив (процедура)
1. Розпакувати архіви у **scratch** (НЕ комітити витягнуте), напр.:
   ```
   for z in docs/_harvest_inbox/*.zip; do
     unzip -o "$z" -d "/tmp/harvest/$(basename "$z" .zip)"
   done
   ```
2. Прогнати по витягнутих `.md` ту саму рубрику, що в kickoff §1
   (HARVEST / PARTIAL / SKIP; фільтр «як система мислить зараз»: жива / витіснена
   сервером / переформулювати). **Скімити суб-агентами**, щоб берегти контекст —
   не вичитувати все підряд.
3. Витягнуте зерно вкласти у майстер-план + `tracking_debt.md` + `DEVELOPMENT_PHILOSOPHY.md`.
4. **Прибрати цю папку** (`_harvest_inbox/`) разом з архівами — відпрацьовані чат-дампи
   у постійному репо не лишаються (конвенція «лише канонічне лишається»).

## Чого НЕ робити
- НЕ комітити розпаковані `.md` у репо (тільки `.zip` тимчасово; витягнуте — у scratch).
- НЕ вважати ці файли джерелом правди — це сире, можливо застаріле під клієнт+Drive;
  кожну ідею перепрочитати крізь серверну модель.
