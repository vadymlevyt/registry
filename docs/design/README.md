# docs/design — макети-еталони дизайн-системи

**Що це.** Окремі самодостатні HTML-макети для **обговорення і проєктування**
візуального шару (палітра, теми, приклади компонентів). Це **не код застосунку** —
файли не входять у `src/`, не потрапляють у Vite-білд і нічого в програмі не змінюють.
Це «ескізи на кальці» поверх реальних значень із коду.

**Навіщо.** Напрямок «уніфікація візуального шару» (див.
`docs/consultations/handoff_2026-06-27_ui_design_unification_state_and_plan.md`).
Перед тим як зводити два набори токенів в один (Рівень 0) і додавати світлу тему
(Рівень 1), корисно **побачити** кольори наочно й ухвалити рішення по точках
розходження (drift).

## Файли

| Файл | Що показує |
|------|-----------|
| `palette.html` | Уся палітра + наочне **порівняння 7 точок drift** (старий набір `App.css` ↔ новий `tokens.css`). Тут ухвалюється «яке значення канонічне». |
| `theme-preview.html` | Жива картка справи з **тумблером темна/світла**. Демонструє «перемкнув тему — перемалювалось усе». Світла тема — чернетка-пропозиція (в застосунку її ще нема). |

## Як дивитись

**Локально, вже зараз** (не потребує платформи): відкрий будь-який `.html`
подвійним кліком у браузері. Значення кольорів у макетах звірені з кодом
станом на 2026-07-01.

**Через Claude Design** (коли платформа доступна): файли позначені коментарем
`<!-- @dsCard group="Foundations" -->` у першому рядку — інструмент бере їх у
галерею дизайн-системи як картки. Синхронізація — по одному файлу, через
навичку `/design-sync`.

## Межі (важливо)

- Значення тут — **дзеркало** коду, не джерело правди. Джерело правди —
  `src/styles/tokens.css` (+ поки що `src/App.css`). Якщо міняється код —
  ці макети оновлюються вручну, не навпаки.
- Світла тема — **пропозиція для обговорення**, не затверджений дизайн.
- Уніфікація ≠ редизайн: макети служать, щоб *зберегти* наявний вигляд і навести
  лад, а не вигадати нову естетику.
