## 4. КАРТА ШЛЯХУ OCR→ТЕКСТ→МЕЖІ→02

```
DPv2 → pipeline.run → streamingExecutor.run → streamFile(кожен PDF):
   chunkManager.planChunks → memoryMonitor.adviseChunkPages (сторінок/чанк, ПО СЕРЕДНЬОМУ байту — див. §C)
   для чанка: materializeChunk(worker splitPdf→_temp) → readChunkBytes
      → processChunk = ocrChunkBytes (DocumentPipelineContext.jsx:110)
          → ocrService.extractText({localBlob}) → documentAi.extract
              → documentAi.js:298 — if (byteLength > 40МБ) throw 'Файл більший за 40 МБ'
   mergeText → streamed text/layout per fileId
стадії: detectBoundaries=triageStage (межі з OCR-тексту) → confirm → extractV3 (processedText) →
   persist=splitDocumentsV3 → writeProcessedArtifacts → writeText02/writeLayout02 (→02_ОБРОБЛЕНІ)
   → [пост-крок cleanFinalizedDocument, якщо cleanForReading]
```
Порожній `getStreamedText(fileId)` → triage бачить порожньо → **1 документ**; extractV3 passthrough → **02 порожнє**. Тобто «1 doc» + «порожнє 02» = **OCR-текст не дійшов**. `EXECUTOR_THREW 40 МБ` = **чанк >40 МБ дійшов до OCR**.
