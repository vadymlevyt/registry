# TASK B доопрацювання — після першого тестування

Адвокат протестував TASK B на ухвалі з 7 фото. Виявлено 4 проблеми які треба виправити.

## ПРОБЛЕМА 1: Дублікати сторінок не виявляються

Серед 7 фото деякі сторінки сфотографовані двічі (з різних ракурсів або повторно). Агент сортування не помітив що це дублікати тієї самої сторінки і вставив обидві у фінальний PDF.

### Що виправити

Розширити imageSortingAgent щоб виявляв дублікати на основі схожості тексту OCR.

Якщо два зображення мають практично ідентичний текст — це дублікати тієї самої сторінки.

Повертати у JSON відповіді додаткове поле `duplicates`:

```json
{
  "order": [0, 1, 2, 3, 4],
  "duplicates": [
    {
      "group": [3, 5],
      "recommended": 3,
      "reason": "Фото 3 чіткіше за фото 5"
    }
  ],
  "warnings": [...],
  "missing": "..."
}
```

Рекомендація на основі:
- Довжини розпізнаного тексту (більше символів зазвичай = чіткіше OCR розпізнав)
- Confidence Document AI якщо є
- Відсутності спотворень

У preview група дублікатів позначається:
- Спільна кольорова рамка (жовта наприклад) навколо обох
- Кращий варіант підсвічений зеленим border додатково
- Текст під ними: "Дублікат — рекомендую залишити цей"
- Кнопка "Залишити рекомендований" одним тапом видаляє інших

## ПРОБЛЕМА 2: Орієнтація не виправляється

Адвокат сфотографував документ повернутим (телефон landscape для portrait сторінки). У фінальному PDF сторінки залишаються повернутими — не обертаються автоматично.

### Що зробити

Діагностика:
- Перевір чи Document AI повертає orientation для цих файлів (можливо для фото з телефону не повертає)
- Перевір чи orientationCorrector реально викликається у pipeline
- Перевір чи Canvas обертання працює коректно
- Додай детальне логування orientation pipeline

Можливі fallback стратегії:
- Якщо Document AI повертає orientation=0 але співвідношення сторін зображення landscape (width > height) і Document AI знайшов вертикально-орієнтований текст — це підозра що фото повернуте
- Або витягувати EXIF orientation з самого зображення перед OCR
- Запропонувати адвокату варіант "повернути" якщо є сумніви через warning

## ПРОБЛЕМА 3: Ручне обертання у preview

Якщо агент не виправив автоматично — адвокат повинен мати можливість виправити сам.

### Що додати

- Кнопка "↻ Повернути" біля кожного thumbnail у preview
- Один тап обертає на 90° за годинниковою стрілкою
- Чотири тапи — повний оборот (повертається до початкового)
- Стан зберігається у preview і застосовується при склейці фінального PDF

## ПРОБЛЕМА 4: Thumbnails занадто маленькі

Адвокат не може розглянути деталі сторінки у маленьких thumbnails 120×140 на планшеті. Не зрозуміло що за сторінка щоб перепорядкувати правильно або перевірити що це правильна сторінка.

### Рішення — кнопки під кожним thumbnail

**Іконка "👁️"** — відкриває попап з збільшеною версією картинки:
- Розмір 600-800 пікселів (читабельний розмір абзаців)
- Попап позиціонується поверх інтерфейсу
- Сусідні thumbnails НЕ рухаються — попап просто вилазить зверху
- Легка тінь навколо попапу щоб виглядало як підняте над поверхнею

Підтримує жести:
- Pinch-to-zoom (два пальці розсунути/стиснути) — збільшити/зменшити масштаб
- Pan (тягнути одним пальцем) — переміщатись по збільшеному зображенню
- Подвійний тап — скинути масштаб до початкового

Використати бібліотеку `react-zoom-pan-pinch` (~5 KB, працює на touch і миші).

Закриття:
- Кнопка "✕" у куті попапу
- Клавіша Escape на десктоп
- НЕ тап поза попапом — інакше випадково закривається коли pan виходить за межі картинки

**Іконка "↻ Повернути"** — обертає thumbnail на 90° за годинниковою стрілкою (Проблема 3).

**Іконка "✕ Видалити"** — видаляє зображення з preview.

**Drag-and-drop за саму картинку** (палець тримає + тягне) — переміщення між позиціями.

Швидкі тапи по кнопках — окремі дії без drag.

### Корисний UX сценарій

Адвокат тапає 👁️ на сторінці N, збільшує кінець тексту через pinch-zoom, читає. Закриває. Тапає 👁️ на сторінці N+1, збільшує початок тексту, читає. Перевіряє чи продовжує думку — якщо ні, переставляє через drag-and-drop.

## ДЕСКТОП ВЕРСІЯ

Адвокат працює переважно з планшета, але деякі сесії з iMac або Windows ПК. UX має бути комфортним на обох.

Адаптації для десктоп:

**Thumbnails більші** — 200×260 пікселів замість 120×140 на планшеті, бо більше місця на екрані. Grid 4-6 колонок замість 3.

**Кнопки під thumbnail** — більші іконки (24×24 пікселів замість 18×18), хover ефект (підсвічування при наведенні миші). Tooltip при hover ("Переглянути", "Повернути", "Видалити").

**Drag-and-drop** — drag handle (іконка ⋮⋮ або ☰) у куті thumbnail для більш очевидного візуального натяку що картинку можна тягнути. На планшеті drag handle опціональний.

**Попап перегляду** — більший розмір 1000-1200 пікселів на десктоп (бо екран дозволяє). Pinch-zoom працює через Ctrl + scroll wheel. Pan через драг мишею.

**Клавіатурні скорочення:**
- Esc — закрити попап
- Стрілки ←/→ у попапі — перейти до попереднього/наступного thumbnail без закриття
- Delete — видалити поточний thumbnail
- R — повернути на 90°
- Ctrl+A — виділити всі thumbnails для масових дій (видалити підозрілі тощо)

**Контекстне меню** — правий клік по thumbnail відкриває меню з усіма діями (Переглянути, Повернути, Видалити, Перейти на N-у позицію).

Це додатково до touch UX, не заміна. Touch працює і на десктопі (якщо є touchscreen) — кожен користувач обирає що зручніше.
